基本类型和包装类型的区别？

  • 用途：除了定义一些常量和局部变量之外，我们在其他地方比如方法参数、对象属性中很少会使用基本类型来定义变量。并且，包装类型可用于泛型，而基本类型不可以。
  • 存储方式：基本数据类型的局部变量存放在 Java 虚拟机栈中的局部变量表中，基本数据类型的成员变量（未被 static 修饰 ）存放在 Java 虚拟机的堆中。包装类型属于对象类型，我们知道几乎所有对象实例都存在于堆中。
  • 占用空间：相比于包装类型（对象类型）， 基本数据类型占用的空间往往非常小。
  • 默认值：成员变量包装类型不赋值就是 null ，而基本类型有默认值且不是 null。
  • 比较方式：对于基本数据类型来说，== 比较的是值。对于包装数据类型来说，== 比较的是对象的内存地址。所有整型包装类对象之间值的比较，全部使用 equals() 方法。

为什么说是几乎所有对象实例都存在于堆中呢？ 这是因为 HotSpot 虚拟机引入了 JIT 优化之后，会对对象进行逃逸分析，如果发现某一个对象并没有逃逸到方法外部，那么就可能通过标量替换来实现栈上分配，而避免堆上分配内存

⚠️ 注意：基本数据类型存放在栈中是一个常见的误区！ 基本数据类型的成员变量如果没有被 static 修饰的话（不建议这么使用，应该要使用基本数据类型对应的包装类型），就存放在堆中。

}

在Java中，使用基本类型和包装类型都有其合适的使用场景。

基本类型（primitive types）通常包括 int、double、boolean、char 等，它们在内存中以原始的二进制形式存储，具有较低的内存消耗和更高的性能。在某些情况下，使用基本类型是更为有效的选择，尤其是在需要高性能和低内存占用的应用程序中。

包装类型（wrapper classes）是对基本类型的对象封装，例如 [...] 等。它们具有更多的功能，包括可以表示 null 值、支持集合和泛型，以及提供了更多的方法。包装类型通常用于需要对象而不是基本类型的情况，例如在集合类中存储基本类型的值或在需要处理 null 值的情况下。

以下是一些情况下的推荐用法：

  1. 在需要处理 null 值的情况下，使用包装类型是更好的选择。例如，在数据库中存储可能为空的字段时，使用包装类型可以表示缺少值。

  2. 在需要与集合类一起使用时，包装类型更方便。集合类通常不支持基本类型，因此如果要将基本类型存储在集合中，必须使用包装类型。

  3. 在进行数学运算或需要低内存消耗和高性能的计算时，基本类型通常更合适。

  4. 在方法参数中，通常使用包装类型更具可读性和灵活性，因为它们支持方法重载和可以接受 null 值。

总之，Java中使用基本类型和包装类型都有其用途，具体选择取决于代码的需求和性能要求。在某些情况下，将基本类型封装为包装类型可以提供更多的功能和灵活性，但也会带来一些性能开销。在其他情况下，使用基本类型可能更加高效。
