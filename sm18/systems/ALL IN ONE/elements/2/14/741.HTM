String#intern 方法有什么作用?

String#intern() 方法是一个用于字符串池（String Pool）的方法。字符串池是 Java 中的一个特性，它是一块保存着字符串对象的内存区域，被设计成共享的。

当你创建一个字符串对象时，Java 会首先检查字符串池中是否已经存在相同值的字符串。如果存在，它将返回池中的实例，而不是创建一个新的对象。如果不存在，它将在字符串池中创建一个新的实例，并返回该实例的引用。

String#intern() 方法的作用是手动将字符串对象加入字符串池。如果调用 intern() 方法的字符串在池中已经存在，它返回池中的引用；如果不在池中，它将该字符串添加到池中，然后返回引用。

这种机制的一个常见用途是确保相同的字符串不会在内存中重复存储，从而节省内存。这在处理大量字符串时，尤其是在涉及很多相同字符串的情况下，可以提高性能和降低内存占用。

以下是一个简单的例子：

String s1 = new String("hello");
String s2 = "hello";

// 下面的 intern() 调用将字符串 "hello" 加入字符串池
String s3 = s1.intern();

System.out.println(s1 == s2);  // false，因为s1是通过new创建的
System.out.println(s2 == s3);  // true，因为s3是从字符串池中返回的引用

需要注意的是，在现代Java中，由于字符串常量池的优化，字符串的创建和处理已经被优化得相当高效，大多数情况下不需要过多地使用 intern() 方法。这个方法通常在你确切知道它的作用并且有明确需求的情况下使用。

String.intern() 是一个 native（本地）方法，其作用是将指定的字符串对象的引用保存在字符串常量池中，可以简单分为两种情况：

  • 如果字符串常量池中保存了对应的字符串对象的引用，就直接返回该引用。
  • 如果字符串常量池中没有保存了对应的字符串对象的引用，那就在常量池中创建一个指向该字符串对象的引用并返回。

示例代码（JDK 1.8） :

// 在堆中创建字符串对象”Java“
// 将字符串对象”Java“的引用保存在字符串常量池中
String s1 = "Java";
// 直接返回字符串常量池中字符串对象”Java“对应的引用
String s2 = s1.intern();
// 会在堆中在单独创建一个字符串对象
String s3 = new String("Java");
// 直接返回字符串常量池中字符串对象”Java“对应的引用
String s4 = s3.intern();
// s1 和 s2 指向的是堆中的同一个对象
System.out.println(s1 == s2); // true
// s3 和 s4 指向的是堆中不同的对象
System.out.println(s3 == s4); // false
// s1 和 s4 指向的是堆中的同一个对象
System.out.println(s1 == s4); //true
