什么是可变长参数？

从 Java5 开始，Java 支持定义可变长参数，所谓可变长参数就是允许在调用方法时传入不定长度的参数。就比如下面的这个 printVariable 方法就可以接受 0 个或者多个参数。

public static void method1(String... args) {
   //......
}

另外，可变参数只能作为函数的最后一个参数，但其前面可以有也可以没有任何其他参数。

public static void method2(String arg1, String... args) {
   //......
}

遇到方法重载的情况怎么办呢？会优先匹配固定参数还是可变参数的方法呢？

答案是会优先匹配固定参数的方法，因为固定参数的方法匹配度更高。

我们通过下面这个例子来证明一下。

/**
 * 微信搜 JavaGuide 回复"面试突击"即可免费领取个人原创的 Java 面试手册
 *
 * @author Guide哥
 * @date 2021/12/13 16:52
 **/
public class VariableLengthArgument {

    public static void printVariable(String... args) {
        for (String s : args) {
            System.out.println(s);
        }
    }

    public static void printVariable(String arg1, String arg2) {
        System.out.println(arg1 + arg2);
    }

    public static void main(String[] args) {
        printVariable("a", "b");
        printVariable("a", "b", "c", "d");
    }
}

输出：

ab
a
b
c
d

另外，Java 的可变参数编译后实际会被转换成一个数组，我们看编译后生成的 class文件就可以看出来了。

public class VariableLengthArgument {

    public static void printVariable(String... args) {
        String[] var1 = args;
        int var2 = args.length;

        for(int var3 = 0; var3 < var2; ++var3) {
            String s = var1[var3];
            System.out.println(s);
        }

    }
    // ......
}

在Java中，当调用一个方法遇到方法重载的情况，编译器会根据提供的参数来选择合适的方法。Java的方法重载解析规则是基于以下原则：

  1. 精确匹配：如果存在与提供的参数精确匹配的方法，那么编译器将选择该方法。这意味着固定参数的方法将比可变参数的方法具有更高的优先级。

  2. 自动类型转换：如果没有找到精确匹配的方法，编译器将查找可以通过自动类型转换（例如，从 int 到 long）匹配参数的方法。

  3. 自动装箱（Autoboxing）：如果还没有找到匹配的方法，编译器将查找可以通过自动装箱和拆箱（例如，从 int 到 Integer）匹配参数的方法。

  4. 可变参数（Varargs）：如果以上规则都没有找到匹配的方法，编译器将选择具有可变参数的方法。这是最后的选择，因为可变参数可以匹配多种参数数量。

根据这些规则，固定参数的方法通常比可变参数的方法具有更高的优先级，因为它们提供了更精确的匹配。然而，如果提供的参数无法精确匹配固定参数方法，编译器将尝试使用自动类型转换或自动装箱来匹配其他方法。只有当以上规则都无法找到匹配的方法时，编译器才会选择可变参数的方法。

下面是一个示例，说明方法重载的解析规则：

class Example {
    void print(int x) {
        System.out.println("int: " + x);
    }

    void print(long x) {
        System.out.println("long: " + x);
    }

    public static void main(String[] args) {
        Example example = new Example();
        int value = 42;
        example.print(value);
    }
}

在这个示例中，print(int x) 方法将被调用，因为提供的参数是 int 类型，它精确匹配了这个方法。如果没有匹配的精确方法，将按照上述规则进行选择。
